# LINE 員工系統角色與介面規則

這份文件是 LINE@ 員工入口的固定規則。調整介面時，必須區分「LINE 聊天室外層圖文選單」與「點進員工系統後的功能列」，不可再混用。

## LINE 聊天室外層圖文選單

A、B、C 三種身分看到相同的三格入口：

1. 查看班表
2. 定位打卡
3. 謎先生備品

外層圖文選單不直接顯示換班。換班權限由員工系統登入後，依 LINE 綁定的員工身分判斷。

## A：兼職人員

員工系統功能列顯示：

- 今日班表
- 班表
- 打卡

兼職人員不顯示換班入口，也不可直接呼叫換班申請功能。

## B：正職人員

員工系統功能列顯示：

- 今日班表
- 班表
- 打卡
- 換班

換班是「正職人員提出申請、管理員確認」的流程，不會由員工自行直接改動正式班表。

## C：管理員

員工系統功能列顯示：

- 今日班表
- 班表
- 打卡
- 換班

管理員可同時保有員工身分並打卡。首頁另外顯示管理後台入口，用於排班、指派人員、審核換班與查看全體資料。

## 其他固定規則

- 「我的申請」不放在員工功能列。
- 具有新人訓練場安排權限的人，從首頁的權限專區進入，不占用一般功能列。
- 灰色班次代表「空場／目前沒有預約」。
- 綠色班次代表「已付訂金／待安排人」。
- 後端仍須再次檢查換班權限，不能只依靠前端隱藏按鈕。

## 安全手機預覽

管理員可從後台「LINE 員工系統」開啟 `line-preview.html`，切換 A 兼職、B 正職、C 管理員，預覽聊天室圖文選單與員工系統功能。預覽使用固定示範資料，不會寫入正式班表、打卡、換班或薪資資料。
